# Уточнение коэффициента sc (refining-sc)

## Входные параметры

Перед запуском нужно определиться с источником изображения и порогами — всё
задаётся флагами скрипта (полная таблица — в «Как запустить»):

- **изображение**: `--hdr` (ENVI-заголовок) и `--dat` (бинарные пиксели);
- **размеры и формат**: `-s/--samples`, `-l/--lines`, `-b/--byte-order`;
- **базовые пороги KEKM**: `-e/--me`, `-a/--ae` (дефолт `0.8`/`0.2`). Это
  пороги NT/ST; OT/AT берут их **удвоенными** (`×2`);
- **что уточнять**: `--methods`, файл статистики `--results`,
  параллелизм `--workers`.

**Тег прогона** = `<image>_<me>_<ae>` — стем файла `--dat` плюс базовые пороги
(например `hsi759_479_0.8_0.2`). По нему именуются и папки, и файл статистики.

## Алгоритм

1. Запустить скрипт. Он прогонит компрессор для нужных кандидатов sc
   **параллельно**, распарсит `Внутренних эталонов: N` и сохранит вывод
   каждого прогона. Папки результатов формируются по тегу прогона и методу:
   `runs/<image>_<me>_<ae>/<METHOD>/output_<sc>.txt` (плюс сводка
   `.../refined.txt`), `<METHOD>` — NT/OT/ST/AT. Так прогоны разных
   изображений/порогов не затирают друг друга.
2. Прочитать информацию по новым запускам из файлов
   `runs/<image>_<me>_<ae>/<METHOD>/output_<sc>.txt` и добавить её в **единый**
   файл статистики `results_top_internal_<image>_<me>_<ae>.txt` (для
   `hsi759_479` с порогами `0.8/0.2` → `results_top_internal_hsi759_479_0.8_0.2.txt`).
   Файл содержит секции всех методов и подаётся на следующий шаг через `--results`.

Если файла статистики ещё нет, скрипт **создаёт шаблон**: по одной seed-строке
на каждый метод с `sc = 1.0` и заведомо огромным числом эталонов. Любой реальный
прогон её побьёт, поэтому уточнение стартует от `sc = 1.0`.


## Как запустить

```bash
# другое изображение и его размеры
python3 .claude/skills/refining-sc/refine_sc.py \
    --hdr .data/other.hdr --dat .data/other.gsd -s 1024 -l 512 -b 0
```

Все параметры скрипта (`--help`):

| Флаг | По умолчанию | Назначение |
|------|--------------|-----------|
| `--methods` | `NT OT ST AT` | какие методы уточнять (один или несколько) |
| `--results` | по тегу прогона | файл статистики; по умолчанию `results_top_internal_<image>_<me>_<ae>.txt` |
| `--workers` | число ядер CPU | число параллельных процессов компрессора |
| `--hdr` | `.data/hsi.hdr` | путь к ENVI-заголовку (абс. или от корня репо) |
| `--dat` | `.data/hsi759_479.gsd` | путь к бинарному изображению |
| `-s, --samples` | `760` | число столбцов |
| `-l, --lines` | `480` | число строк |
| `-b, --byte-order` | `0` | порядок байт (0=LE, 1=BE) |
| `-e, --me` | `0.8` | базовый порог 1 уровня % (NT/ST; OT/AT берут ×2) |
| `-a, --ae` | `0.2` | базовый порог 2 уровня % (NT/ST; OT/AT берут ×2) |

Базовые `me`/`ae` относятся к NT/ST; OT/AT автоматически работают на удвоенных
порогах (`1.6`/`0.4` при дефолте).

Скрипт сам ждёт завершения всех процессов уровня. Перед запуском проект должен быть собран
(`.build/hsi_compressor`).
