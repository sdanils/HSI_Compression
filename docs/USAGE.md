# Как работать с программой

Руководство по практическому использованию **HSI Compressor** — от сборки до
типовых сценариев сжатия и восстановления.

См. также: [процессы сжатия и декомпрессии](COMPRESSION.md),
[математическое обоснование](MATH.md), [скиллы проекта](SKILLS.md).

---

## Сборка

Внешних зависимостей нет — нужны только стандартные инструменты (C++17,
CMake ≥ 3.10). PNG-запись обеспечивает встроенная `include/stb_image_write.h`.

```bash
# Linux (Ubuntu/Debian) — установка инструментов сборки
sudo apt update && sudo apt install build-essential cmake

# Конфигурация и сборка
cmake -S . -B .build
cmake --build .build
```

Бинарь появится по пути `.build/hsi_compressor`.

> При удалении или переименовании `.cpp`/`.h` пересоберите проект с нуля
> (`rm -rf .build`, затем снова `cmake -S . -B .build`): CMake собирает список
> файлов через `GLOB_RECURSE`, и он фиксируется на этапе конфигурации.

## Подкоманды

Интерфейс построен на **подкомандах**: первый аргумент выбирает режим.

```
./hsi_compressor compress   <hdr_path> <dat_path> [OPTIONS]
./hsi_compressor decompress [hdr_path]            [OPTIONS]
```

| Режим | Назначение | Обязательные аргументы |
|-------|-----------|------------------------|
| `compress`   | Сжатие изображения в пару GSD-файлов | `.hdr` и `.gsd` (оба) |
| `decompress` | Восстановление `.dat` из GSD-файлов | — (`.hdr` опционален) |

В режиме `compress` оба позиционных пути обязательны (значений по умолчанию нет).
В режиме `decompress` путь к `.hdr` нужен только ради `byte_order` (в GSD он не
хранится); по умолчанию `/.data/hsi.hdr`. Размеры (`samples`/`lines`/`bands`)
берутся из самих GSD-файлов.

---

## Параметры командной строки

| Параметр | Описание | По умолчанию |
|----------|----------|-------------|
| `hdr_path` | Путь к `.hdr` файлу | `/.data/hsi.hdr` (только decompress) |
| `dat_path` | Путь к `.gsd` файлу (только compress) | — (обязателен) |
| `-m, --method <1-4>` | Метод КЭКМ: 1=НП, 2=ОП, 3=МП, 4=АП | `1` (НП) |
| `-e, --me <%>` | Порог 1-го уровня, % от нормы пикселя | `5.0` |
| `-a, --ae <%>` | Порог 2-го уровня, % от нормы пикселя | `2.0` |
| `-S, --shift <0\|1>` | Смещённое размещение нового эталона | `1` (включено) |
| `-c, --sc <x>` | Коэффициент шага решётки: d = sc·δ, 1 ≤ sc ≤ 2 | `1.0` |
| `-r, --rgb <0\|1>` | Генерировать RGB-превью | `1` (включено) |
| `-w, --write <0\|1>` | Записывать выходные GSD-файлы | `1` (включено) |
| `-d, --dat-out <file>` | Выходной `.dat` (только decompress) | `restored.dat` |
| `-p, --preview <file>` | Имя файла превью оригинала | `preview.png` |
| `-P, --restored <file>` | Имя файла превью восстановленного | `restored_preview.png` |
| `-o, --std <file>` | Выходной файл эталонов | `standarts.gsd` |
| `-i, --img <file>` | Выходной файл сжатого изображения | `image.gsd` |
| `-s, --samples <N>` | Переопределить число столбцов из заголовка | — |
| `-l, --lines <N>` | Переопределить число строк из заголовка | — |
| `-b, --byte-order <N>` | Переопределить порядок байт (0=LE, 1=BE) | — |

## Входные файлы

| Файл | Формат | Описание |
|------|--------|----------|
| `.hdr` | Текст (ENVI) | Метаданные: размеры, порядок байт, длины волн |
| `.gsd` | Бинарный (int16, BIP) | Спектральные значения всех пикселей |

Пример `.hdr`:

```
samples = 800
lines   = 600
bands   = 128
byte order = 0
wavelength = { 400.0, 403.2, 406.4, ... }
```

`BIP` (Band Interleaved by Pixel) — порядок, в котором значения всех полос
одного пикселя лежат подряд, затем идёт следующий пиксель.

---

## Выходные файлы

| Файл | Когда создаётся | Описание |
|------|-----------------|----------|
| `standarts.gsd` | `compress`, `--write 1` | Словарь эталонов (бинарный) |
| `image.gsd` | `compress`, `--write 1` | Карта сжатых пикселей (ссылка + параметры) |
| `preview.png` | `compress`, `--rgb 1` | RGB-превью оригинала |
| `restored_preview.png` | `compress`, `--rgb 1` | RGB-превью восстановленного (та же тональная шкала) |
| `restored.dat` | `decompress` | Восстановленная матрица пикселей (int16, BIP) |

Подробные структуры GSD-файлов — в [COMPRESSION.md](COMPRESSION.md#форматы-выходных-файлов).

## Типовые сценарии

```bash
# Базовое сжатие (метод НП по умолчанию)
./hsi_compressor compress data/scene.hdr data/scene.gsd

# Выбор метода АП (аффинная инвариантность)
./hsi_compressor compress data/scene.hdr data/scene.gsd --method 4

# Жёсткое сжатие: меньше эталонов, ниже качество
./hsi_compressor compress data/scene.hdr data/scene.gsd --me 15.0 --ae 7.0

# Высокое качество: много эталонов, минимум потерь
./hsi_compressor compress data/scene.hdr data/scene.gsd --me 1.0 --ae 0.5 -m 4

# Отключить смещённое размещение (эталон = сам пиксель)
./hsi_compressor compress data/scene.hdr data/scene.gsd --shift 0

# Активировать смещение на полный шаг решётки (d = 2δ)
./hsi_compressor compress data/scene.hdr data/scene.gsd --sc 2.0

# Без превью и без записи GSD (только подсчёт эталонов в stdout)
./hsi_compressor compress data/scene.hdr data/scene.gsd --rgb 0 --write 0

# Декомпрессия: восстановить .dat из GSD-файлов (.hdr нужен для byte_order)
./hsi_compressor decompress data/scene.hdr \
    --std standarts.gsd --img image.gsd --dat-out out.dat

# Переопределить метаданные заголовка
./hsi_compressor compress data/scene.hdr data/scene.gsd -s 1024 -l 512 -b 1
```

## Как параметры влияют на качество и размер

Пороги `--me` и `--ae` — это **проценты от энергии/дисперсии самого пикселя**
(адаптивные, см. [MATH.md](MATH.md#адаптивные-пороги)). Чем выше процент — тем
шире зона каждого эталона, тем меньше эталонов и сильнее сжатие, но ниже
точность.

```
Высокие пороги (--me 15 --ae 8):
  + мало эталонов  → сильное сжатие
  − заметные потери качества

Низкие пороги (--me 1 --ae 0.3):
  + высокое качество восстановления
  − много эталонов → слабое сжатие
```

- `--me` (main error) управляет числом **основных** эталонов — крупных классов
  материалов (трава, асфальт, вода).
- `--ae` (additional error) управляет числом **под-эталонов** — мелких вариаций
  внутри класса.
- `--method` АП (`-m 4`) при тех же порогах «объясняет» больше вариаций
  (смещение + масштаб), поэтому требует меньше эталонов при той же точности.
- `--sc` (1 ≤ sc ≤ 2) сдвигает новые эталоны ближе к идеальной решётке покрытия,
  уменьшая дублирование зон. При `sc = 1` механизм смещения геометрически
  неактивен — нужно задать `--sc > 1`. Уточнение оптимального `sc` —
  см. скилл [`refining-sc`](SKILLS.md#refining-sc--уточнение-коэффициента-sc).

## Пример сеанса работы

```
$ ./hsi_compressor compress scene.hdr scene.gsd --method 4 --me 3.0 --ae 1.0 --sc 1.5

[debug] hdr=scene.hdr dat=scene.gsd
[debug] samples=800 lines=600 bands=128 byte_order=0 method=4
[debug] main_error_pct=3% additional_error_pct=1% shift=1 step_coef=1.5 rgb=1 write_gsd=1
Главных эталонов: 52  Внутренних эталонов: 1340  (sc: 1.5)
```

Записаны: `preview.png`, `standarts.gsd`, `image.gsd`, `restored_preview.png`.
«Главных эталонов» — число основных классов, «Внутренних эталонов» — общее число
под-эталонов (включая базовые).
