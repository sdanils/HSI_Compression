# Математическое обоснование

Полный вывод формул, на которых стоит алгоритм: четыре метода КЭКМ как задачи
минимизации ошибки восстановления, смысл адаптивных порогов и геометрия
смещённого размещения эталонов.

Алгоритмическая сторона (что за чем идёт в коде) — в
[COMPRESSION.md](COMPRESSION.md). Практика — в [USAGE.md](USAGE.md).

---

## Обозначения

Спектр пикселя и эталон — векторы длины `n = bands`. Пусть `e` — эталон,
`y` — сжимаемый пиксель. Все статистики берутся **по полосам** (по `n`
компонентам вектора):

| Обозначение | Определение | В коде |
|-------------|-------------|--------|
| `μ(x)`  | среднее `(1/n)·Σ xᵢ` | `mean()` |
| `M²(x)` | второй момент `(1/n)·Σ xᵢ²` | `moment2()` |
| `D(x)`  | дисперсия `(1/n)·Σ (xᵢ − μ(x))² = M²(x) − μ(x)²` | `disp()` |
| `M(ey)` | кросс-момент `(1/n)·Σ eᵢ·yᵢ` | `cross()` |
| `cov(e,y)` | ковариация `(1/n)·Σ (eᵢ−μ(e))(yᵢ−μ(y)) = M(ey) − μ(e)·μ(y)` | `cov()` |

> Здесь `M²(x)` — это **второй момент** (среднее квадратов), а не квадрат
> среднего. Так принято в коде (`moment2`).

Все четыре метода восстанавливают пиксель из эталона одним и тем же **аффинным**
преобразованием с общими для всех полос коэффициентами:

```
ŷᵢ = k·eᵢ + d
```

Различаются методы тем, **какие коэффициенты свободны** (могут подстраиваться под
пиксель), а какие зафиксированы. Это и задаёт инвариантность метода.

Мера расхождения — средняя квадратичная ошибка восстановления на полосу:

```
ε(k, d) = (1/n)·Σ (yᵢ − k·eᵢ − d)²
```

Каждый метод минимизирует `ε` по своим свободным параметрам; минимум `ε` и есть
`epsilon` в `kekm_result`, а оптимальные `k, d` — это `k_m` и `delta_y`.

## Методы КЭКМ: вывод формул

### НП — неинвариантный (`k = 1`, `d = 0`)

Оба коэффициента зафиксированы: эталон используется как есть, `ŷ = e`. Свободных
параметров нет, минимизировать нечего:

```
ε = (1/n)·Σ (yᵢ − eᵢ)²
  = (1/n)·Σ yᵢ² − 2·(1/n)·Σ eᵢyᵢ + (1/n)·Σ eᵢ²
  = M²(y) + M²(e) − 2·M(ey)
```

→ `epsilon = M²(e) + M²(y) − 2·M(ey)`, `k_m = 1`, `delta_y = 0`.

Пиксель должен быть почти идентичен эталону — никакой компенсации
яркости/масштаба нет. Самый быстрый метод.

### ОП — инвариантный к ортогональным преобразованиям (`k = 1`, `d` свободно)

Допускается общий аддитивный сдвиг яркости `d`. Модель `ŷ = e + d`.
Минимизируем по `d`:

```
∂ε/∂d = −2·(1/n)·Σ (yᵢ − eᵢ − d) = 0
   ⟹  d = μ(y) − μ(e)
```

→ `delta_y = μ(y) − μ(e)` (выравнивание средних). Подставляя обратно, ошибка
выражается через центрированные векторы:

```
ε = (1/n)·Σ [ (yᵢ − μ(y)) − (eᵢ − μ(e)) ]²
  = D(y) + D(e) − 2·cov(e,y)
```

→ `epsilon = D(e) + D(y) − 2·cov(e,y)`, `k_m = 1`.

Смысл: тень, делающая весь спектр темнее на константу, не штрафуется.

### МП — инвариантный к масштабированию (`k` свободно, `d = 0`)

Допускается общий множитель `k` (другая освещённость). Модель `ŷ = k·e`.
Минимизируем по `k`:

```
∂ε/∂k = −2·(1/n)·Σ eᵢ(yᵢ − k·eᵢ) = 0
   ⟹  k = Σ eᵢyᵢ / Σ eᵢ² = M(ey) / M²(e)
```

→ `k_m = M(ey) / M²(e)`. Подставляя:

```
ε = M²(y) − 2k·M(ey) + k²·M²(e)
  = M²(y) − M(ey)²/M²(e)             (после k = M(ey)/M²(e))
```

→ `epsilon = M²(y) − M²(ey)/M²(e)`, `delta_y = 0`.

**Вырожденный случай.** Если `M²(e) ≤ 0` (нулевой эталон), масштаб не определён.
Код откатывается на НП: `k_m = 1`, `delta_y = 0`, `epsilon = M²(y)` (это и есть
формула НП при `e ≡ 0`).

### АП — инвариантный к аффинным преобразованиям (`k` и `d` свободны)

Самый гибкий: и сдвиг, и масштаб. Модель `ŷ = k·e + d` — обычная линейная
регрессия `y` на `e`. Из `∂ε/∂d = 0` получаем `d = μ(y) − k·μ(e)`; подставив в
`∂ε/∂k = 0`:

```
k = cov(e,y) / D(e)
d = μ(y) − k·μ(e)
```

→ `k_m = cov(e,y)/D(e)`, `delta_y = μ(y) − k_m·μ(e)`. Остаточная ошибка — это
дисперсия `y`, не объяснённая регрессией:

```
ε = D(y) − cov(e,y)²/D(e) = D(y)·(1 − r²),   r² = cov²(e,y) / (D(e)·D(y))
```

→ `epsilon = D(y) − cov²(e,y)/D(e)`.

Здесь `r` — коэффициент корреляции эталона и пикселя: чем сильнее линейная связь,
тем меньше остаток. Даёт лучшее сжатие при той же точности.

**Вырожденный случай.** Если `D(e) ≤ 0` (константный эталон), масштаб не
определён (`cov` тоже `= 0`). Откат на ОП: `k_m = 1`, `delta_y = μ(y) − μ(e)`,
`epsilon = D(y)`.

### Сводка

| Метод | Свободно | `k_m` | `delta_y` | `epsilon` |
|-------|----------|-------|-----------|-----------|
| НП | — | `1` | `0` | `M²(e) + M²(y) − 2·M(ey)` |
| ОП | `d` | `1` | `μ(y) − μ(e)` | `D(e) + D(y) − 2·cov(e,y)` |
| МП | `k` | `M(ey)/M²(e)` | `0` | `M²(y) − M²(ey)/M²(e)` |
| АП | `k, d` | `cov(e,y)/D(e)` | `μ(y) − k_m·μ(e)` | `D(y) − cov²(e,y)/D(e)` |

Обратное преобразование при восстановлении одно для всех методов:
`ŷᵢ = round(k_m·eᵢ + delta_y)` (см. [COMPRESSION.md](COMPRESSION.md#универсальная-формула-восстановления)).

## Адаптивные пороги

Порог принятия пикселя в эталон масштабируется под сам пиксель:

```
threshold = (error_pct / 100) · pixel_norm(y)
```

где `pixel_norm(y) = M²(y)` для НП/МП и `= D(y)` для ОП/АП. Почему именно эти
величины — они равны **максимальной ошибке метода**, когда эталон бесполезен:

- **НП/МП:** при `e ≡ 0` (или `M(ey) = 0` для МП) минимальная `ε = M²(y)` — это
  вся энергия пикселя. То есть `M²(y)` — ошибка «восстановления нулём».
- **ОП/АП:** при константном `e` (или `cov = 0` для АП) остаётся `ε = D(y)` —
  вся вариация пикселя относительно его среднего.

Поэтому `ε / pixel_norm(y)` — это **относительная** ошибка восстановления, а
`error_pct` — её допустимый процент. Один и тот же `--me 5` означает «ошибка не
больше 5 %» и для яркого, и для тёмного пикселя — отсюда устойчивость алгоритма к
перепадам освещённости сцены без подстройки параметров.

`--me` задаёт порог на уровне 1 (число классов), `--ae` — на уровне 2 (детали
внутри класса).

## Геометрия покрытия и смещение эталона

### Линейная шкала и радиус зоны

Введём евклидову длину невязки в пространстве полос:

```
ρ = √(n·ε)
```

Так как `n·ε = Σ (yᵢ − ŷᵢ)²`, величина `ρ = ‖y − ê‖` — это обычное евклидово
расстояние от пикселя до **выровненного** эталона `ê = k_m·e + delta_y`. В этой
шкале порог становится **радиусом зоны** эталона:

```
r = √(n·threshold)
```

Пиксель принимается эталоном тогда и только тогда, когда `ρ ≤ r`. То есть каждый
эталон «накрывает» шар радиуса `r` вокруг себя (в выровненных координатах).

### Зачем смещать эталон

Если новый эталон поставить **прямо в пиксель** `p`, его шар радиуса `r` сильно
перекрывается с шарами уже существующих соседей: общая площадь покрытия
дублируется, а между зонами остаются непокрытые «карманы», под которые потом
придётся заводить лишние эталоны. Идеальное покрытие пространства шарами — это
**решётка** с шагом порядка `r`, а не плотная куча в одной точке. Смещение
двигает новый эталон к узлу такой решётки.

### Целевой шаг и формула отталкивания

Реализация — `make_shifted_etalon` ([src/etalon_shift.cpp](../src/etalon_shift.cpp)).
Целевой шаг задаётся коэффициентом `sc` (`--sc`) в шкале `ε`, что в линейной
шкале даёт целевое расстояние `target_ρ`:

```
target_ε = sc · threshold
target_ρ = √(n · target_ε) = √(sc) · r
```

За один проход по соседям-эталонам: для каждого соседа `eᵢ`, который попадает в
целевую зону (`0 < εᵢ < target_ε`, то есть `ρᵢ < target_ρ`), текущий снимок
пикселя `y` **отталкивается** от выровненного соседа `êᵢ = k_m·eᵢ + delta_y`
вдоль единичного направления `(y − êᵢ)/ρᵢ` на величину `target_ρ − ρᵢ`:

```
ρᵢ    = √(n·εᵢ)
shift = target_ρ − ρᵢ
y  ←  y + (y − êᵢ)/ρᵢ · shift
```

Для одного соседа это ровно выводит `y` на расстояние `target_ρ` от него; при
нескольких соседях сдвиги накапливаются (аккумулятор в `double` для полной
точности между итерациями, сравнения — по округлённому снимку int16). В
результате новый эталон оказывается примерно на шаге `target_ρ` от соседей —
ближе к узлу решётки, чем исходный пиксель.

### Выравнивание `ê` — почему формула одна на все методы

Отталкивание идёт не от самого `eᵢ`, а от **метод-выровненного** `êᵢ = k_m·eᵢ + delta_y`
— того же, что используется при восстановлении. Благодаря этому единая формула
корректна для всех четырёх методов:

| Метод | `ê` |
|-------|-----|
| НП | `e` (k=1, d=0) |
| ОП | `e + delta_y` (центрирование) |
| МП | `k_m·e` (масштабирование) |
| АП | `k_m·e + delta_y` (оба) |

### Границы и якорь

- При `sc = 1` целевая зона совпадает с зоной приёма: все соседи нового эталона
  по построению **дальше** `r` (иначе пиксель был бы распознан и эталон не
  создавался), поэтому ни один сосед не даёт вклад — смещение геометрически
  **неактивно**. Чтобы оно заработало, нужно `sc > 1` (по теории до 2).
- На **уровне 2** смещение ограничено **якорем**: кандидат-сдвиг принимается,
  только если смещённая копия остаётся в зоне основного эталона
  (`pixel_distance(anchor, y) ≤ main_threshold`). Это не даёт под-эталону уйти из
  своего класса. На уровне 1 якоря нет.
- После создания эталона `set_match` записывает **фактическое**
  `pixel_distance(e_new, p)` в карту пикселя — поэтому декомпрессия точно
  восстанавливает исходный пиксель из смещённой копии независимо от величины
  сдвига.

Подбор оптимального `sc` (минимизирующего число под-эталонов) автоматизирован
скиллом [`refining-sc`](SKILLS.md#refining-sc--уточнение-коэффициента-sc).
