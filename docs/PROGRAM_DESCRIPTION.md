# HSI Compressor — Подробное описание программы

## Что такое программа

**HSI Compressor** — это инструмент командной строки для сжатия гиперспектральных изображений (HSI — Hyperspectral Imaging). Программа реализует оригинальный алгоритм сжатия на основе метода квантования спектральных эталонов (КЭКМ — Квантование Эталонных Кластеров Многомерных данных).

Гиперспектральное изображение — это не обычная фотография: каждый пиксель содержит сотни значений (спектральных полос), описывающих интенсивность отражения света на разных длинах волн. Такие данные используются в дистанционном зондировании Земли, медицинской диагностике, пищевой и фармацевтической промышленности, геологоразведке.

---

## Принцип работы

### Общая идея

Алгоритм основан на наблюдении: пиксели гиперспектрального изображения часто похожи друг на друга — один и тот же тип поверхности (трава, асфальт, вода) даёт похожие спектральные профили. Вместо того чтобы хранить каждый пиксель целиком, программа:

1. Строит словарь **эталонов** — характерных спектральных векторов.
2. Каждый пиксель заменяется **ссылкой на ближайший эталон** + небольшой поправкой для точного восстановления.

Это аналогично тому, как словарь слов меньше, чем весь текст книги.

### Двухуровневая структура эталонов

Эталоны организованы иерархически:

```
Основной эталон 0 ─┬─ Под-эталон 0 (базовый)
                   ├─ Под-эталон 1
                   └─ Под-эталон 2

Основной эталон 1 ─┬─ Под-эталон 0 (базовый)
                   └─ Под-эталон 1
...
```

- **Уровень 1 (основные эталоны):** Новый основной эталон создаётся, если пиксель слишком далёк от всех существующих. Порог задаётся параметром `--me` (main error, % от нормы пикселя).
- **Уровень 2 (под-эталоны):** Внутри подходящей группы новый под-эталон создаётся, если пиксель достаточно отличается от всех под-эталонов группы. Порог — `--ae` (additional error, %).

### Адаптивные пороги

Пороги сравнения не фиксированы, а **масштабируются под каждый пиксель**:

```
threshold = (error_pct / 100.0) × pixel_norm(pixel)
```

Где `pixel_norm` — мера "энергии" пикселя (второй момент для NT/ST, дисперсия для OT/AT). Это позволяет одинаково хорошо обрабатывать яркие и тёмные участки изображения.

### Процесс сжатия (пошагово)

```
Входной .gsd файл (int16, BIP-интерлив)
    ↓
load_hsi_data()  — загрузка пикселей в матрицу [lines×samples][bands]
    ↓
save_rgb_image() — сохранение превью оригинала (preview.png)
    ↓
compression()    — итерация по всем пикселям:
    │  для каждого пикселя:
    │    1. Если >50% значений отрицательны → заменить нулевым вектором
    │    2. check_pixel():
    │         Уровень 1: найти ближайший основной эталон
    │         Если epsilon > main_threshold → add_standart() (новый основной)
    │         Иначе Уровень 2: найти ближайший под-эталон
    │         Если epsilon > add_threshold → add_internal_standart()
    │         Иначе → сохранить ref_index + kekm_result {epsilon, delta_y, k_m}
    │
save_standarts_gsd()        — сохранение эталонов (standarts.gsd)
save_compressed_image_gsd() — сохранение карты пикселей (image.gsd)
    ↓
decompress()     — восстановление из памяти
    │  для каждого пикселя:
    │    ŷ[b] = round(k_m × ref[b] + delta_y)
    │
save_rgb_image() — сохранение превью восстановленного (restored_preview.png)
```

## Методы КЭКМ (4 варианта)

Ядро алгоритма — функция измерения "расстояния" между спектральными векторами. Реализованы четыре метода с разными свойствами инвариантности:

| Метод | Флаг | Инвариантность | Формула epsilon | Восстановление |
|-------|------|---------------|-----------------|----------------|
| **НП** (неинвариантная) | `-m 1` | — | `M²(e) + M²(y) − 2·M(ey)` | `ŷ = ref` (прямая копия) |
| **ОП** (ортогональная) | `-m 2` | Ортогональные преобразования | `D(e) + D(y) − 2·cov(e,y)` | `ŷ = ref + delta_y` |
| **МП** (масштабная) | `-m 3` | Масштабирование | `M²(y) − M²(ey)/M²(e)` | `ŷ = k_m × ref` |
| **АП** (аффинная) | `-m 4` | Аффинные преобразования | `D(y) − cov²(e,y)/D(e)` | `ŷ = k_m × ref + delta_y` |

**Что это означает практически:**

- **НП** — самый быстрый и простой. Пиксель должен быть почти идентичен эталону.
- **ОП** — допускает глобальный сдвиг яркости (например, тень делает весь спектр темнее).
- **МП** — допускает масштабирование (пиксель похож на эталон, но умноженный на константу — другая освещённость).
- **АП** — наиболее гибкий: допускает и смещение, и масштабирование одновременно. Даёт наилучшее сжатие при схожей точности.

## Входные и выходные данные

### Входные файлы

| Файл | Формат | Описание |
|------|--------|----------|
| `.hdr` | Текст (ENVI) | Метаданные: размер изображения, порядок байт, длины волн |
| `.gsd` | Бинарный (int16, BIP) | Пиксельные данные — спектральные значения всех пикселей |

Формат `.hdr` (пример):
```
samples = 800
lines   = 600
bands   = 128
byte order = 0
wavelength = { 400.0, 403.2, 406.4, ... }
```

### Выходные файлы

| Файл | Описание |
|------|----------|
| `standarts.gsd` | Бинарный файл эталонов: `[int32 total][int32 bands][int16 × total × bands]` |
| `image.gsd` | Карта сжатых пикселей: `[int32 samples][int32 lines]` + на каждый пиксель `[int32 ref_index][double epsilon][double delta_y][double k_m]` |
| `preview.png` | RGB-превью оригинального изображения |
| `restored_preview.png` | RGB-превью восстановленного изображения (визуальная оценка качества) |
| `standarts.txt` | Текстовый вариант эталонов (устаревший) |
| `image.txt` | Текстовый вариант карты пикселей (устаревший) |

### Превью (RGB-визуализация)

Программа автоматически выбирает три канала, ближайшие к длинам волн:
- **Красный:** 660 нм
- **Зелёный:** 550 нм
- **Синий:** 470 нм

Применяется **гистограммное выравнивание** по каждому каналу независимо — это позволяет увидеть детали даже при неравномерной освещённости.

## Параметры командной строки

```
./hsi_compressor [hdr_path dat_path] [OPTIONS]
```

| Параметр | Описание | По умолчанию |
|----------|----------|-------------|
| `hdr_path` | Путь к .hdr файлу | `/.data/hsi.hdr` |
| `dat_path` | Путь к .gsd файлу | `/.data/hsi.gsd` |
| `-m, --method <1-4>` | Метод КЭКМ: 1=НП, 2=ОП, 3=МП, 4=АП | `1` (НП) |
| `--me <%>` | Порог 1-го уровня (% от нормы пикселя) | `5.0` |
| `--ae <%>` | Порог 2-го уровня (% от нормы пикселя) | `2.0` |
| `-s, --samples <N>` | Переопределить число столбцов из заголовка | — |
| `-l, --lines <N>` | Переопределить число строк из заголовка | — |
| `-b, --byte-order <N>` | Переопределить порядок байт | — |

### Примеры использования

```bash
# Запуск с настройками по умолчанию
./hsi_compressor

# Указание файлов + выбор метода АП
./hsi_compressor data/scene.hdr data/scene.gsd --method 4

# Жёсткое сжатие (меньше эталонов, ниже качество)
./hsi_compressor data/scene.hdr data/scene.gsd --me 15.0 --ae 7.0

# Высокое качество (много эталонов, мало потерь)
./hsi_compressor data/scene.hdr data/scene.gsd --me 1.0 --ae 0.5 -m 4

# Переопределение метаданных заголовка
./hsi_compressor data/scene.hdr data/scene.gsd -s 1024 -l 512
```

### Влияние параметров на качество и степень сжатия

```
Высокие пороги (--me 15 --ae 8):
  ✔ Мало эталонов → сильное сжатие
  ✗ Значительные потери качества

Низкие пороги (--me 1 --ae 0.3):
  ✔ Высокое качество восстановления
  ✗ Много эталонов → слабое сжатие

Метод АП (-m 4) при тех же порогах:
  ✔ Лучше "объясняет" вариации → меньше эталонов нужно
  → Более эффективное сжатие при том же качестве
```

## Восстановление (декомпрессия)

Восстановление реализовано тремя способами:

### 1. Из памяти (основной режим)
```cpp
int16_t** restored = decompress(&comp_data, &header);
```
Используется сразу после сжатия — нет I/O накладных расходов.

### 2. Из текстовых файлов (устаревший)
```cpp
int16_t** restored = decompress_from_files("standarts.txt", "image.txt");
```

### 3. Из бинарных GSD-файлов (автономный режим)
```cpp
int16_t** restored = decompress_from_gsd_files("standarts.gsd", "image.gsd");
```
Позволяет восстановить изображение без исходных данных — только из файлов сжатия.

**Универсальная формула восстановления:**
```
ŷ[b] = round(k_m × ref[b] + delta_y)
```
- НП: k_m=1, delta_y=0 → точная копия эталона
- ОП: k_m=1 → эталон + смещение по среднему
- МП: delta_y=0 → масштабированный эталон
- АП: оба параметра → полное аффинное восстановление

## Архитектура кода

### Структура модулей

```
src/
├── main.cpp            — точка входа, разбор аргументов
├── read_hrd_file.cpp   — чтение ENVI .hdr заголовка
├── load_hsi_data.cpp   — загрузка бинарных .gsd данных
├── compression.cpp     — основной цикл сжатия
├── check_pixel.cpp     — двухуровневое сравнение пикселя с эталонами
├── kekm.cpp            — четыре метода КЭКМ + вспомогательные функции
├── decompress.cpp      — восстановление из памяти и файлов
├── save.cpp            — запись GSD и текстовых файлов
├── save_rgb.cpp        — конвертация в PNG с гистограммным выравниванием
├── free_memory.cpp     — освобождение памяти
└── print_pixel.cpp     — отладочный вывод пикселя

include/
├── kekm.h              — типы kekm_method, kekm_result; функции КЭКМ
├── compressed_image.h  — структура compressed_image (3D эталоны + карта)
├── standart_data.h     — структура standart_data (ref_index + kekm_result)
├── hsi_header.h        — структура hsi_header (метаданные изображения)
├── compression_settings.h — параметры сжатия (пороги, метод)
└── functions.h         — зонтичный заголовок (включает все модули)
```

### Ключевые структуры данных

```cpp
// Метаданные изображения
struct hsi_header {
    int samples;       // ширина (столбцы)
    int lines;         // высота (строки)
    int bands;         // число спектральных полос
    int byte_order;    // порядок байт
    float* wavelengths; // длины волн [bands]
};

// Параметры сжатия
struct compression_settings {
    double main_error_pct;       // порог 1-го уровня (%)
    double additional_error_pct; // порог 2-го уровня (%)
    kekm_method method;          // метод КЭКМ
};

// Результат КЭКМ-сравнения
struct kekm_result {
    double epsilon;  // оценка близости (чем меньше — тем лучше)
    double delta_y;  // смещение (для ОП, АП)
    double k_m;      // коэффициент масштабирования (для МП, АП)
};

// Сжатое изображение
struct compressed_image {
    int16_t*** hsi_standarts; // [основной][под-эталон][полоса]
    int num_ref;              // число основных эталонов
    int* ref_counts;          // число под-эталонов у каждого основного
    standart_data** image;    // результат для каждого пикселя
    int size;                 // число пикселей
};
```

## Преимущества программы

### 1. Физически обоснованный алгоритм
Алгоритм использует **спектральное расстояние**, а не обобщённые математические преобразования (DCT, вейвлеты). Это означает, что эталоны соответствуют реальным типам поверхностей — алгоритм "понимает" физический смысл данных.

### 2. Адаптивность к содержимому сцены
Пороги масштабируются под каждый пиксель через его норму — программа одинаково хорошо обрабатывает яркие (высокоотражающие) и тёмные (поглощающие) участки изображения. Нет необходимости настраивать параметры под конкретный тип сцены.

### 3. Четыре метода сравнения — для разных условий съёмки
- Снимки с равномерным освещением → метод НП (быстрее)
- Съёмка при разных углах освещения → метод МП или АП
- Изображения с тенями и засветками → метод АП (максимальная гибкость)

### 4. Регулируемый компромисс качество/сжатие
Два параметра (`--me`, `--ae`) позволяют плавно управлять сжатием — от практически без потерь до агрессивного уменьшения размера.

### 5. Иерархическая структура эталонов
Двухуровневое дерево эффективнее, чем плоский словарь: быстрый первый уровень отсеивает несовместимые классы, второй уровень дообрабатывает детали. Это снижает число сравнений с O(N) до O(классы + детали).

### 6. Встроенная визуализация
Программа автоматически генерирует PNG-превью оригинала и восстановленного изображения с гистограммным выравниванием. Можно мгновенно оценить качество сжатия визуально, не требуя дополнительных инструментов.

### 7. Автономное восстановление
Файлы `standarts.gsd` и `image.gsd` самодостаточны — восстановление возможно без исходных данных и без исходного заголовка. Это важно для архивного хранения.

### 8. Эффективное использование памяти
Динамическое расширение массивов (realloc с удвоением ёмкости) минимизирует число аллокаций. Все ресурсы освобождаются явно.

### 9. Обработка артефактов данных
Пиксели с более чем половиной отрицательных значений (артефакты датчика) автоматически заменяются нулевым вектором — это предотвращает создание паразитных эталонов из шумовых данных.

### 10. Поддержка бинарного и текстового форматов
Бинарные GSD-файлы — компактны и быстры; текстовые TXT-файлы — читаемы человеком и совместимы с внешними инструментами. Оба режима поддерживаются одновременно.
