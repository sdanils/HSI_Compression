# Процессы сжатия и декомпрессии

Описание того, **как именно** программа сжимает гиперспектральное изображение и
восстанавливает его обратно — поток данных, структуры, форматы файлов.

Формулы и их вывод вынесены в [математическое обоснование](MATH.md). Здесь —
алгоритмическая сторона: что за чем идёт и где это в коде.

См. также: [как работать с программой](USAGE.md), [скиллы проекта](SKILLS.md).

---

## Общая идея

Пиксели гиперспектрального изображения часто похожи: один и тот же тип
поверхности даёт похожий спектральный профиль. Вместо хранения каждого пикселя
целиком программа строит **словарь эталонов** (характерных спектров) и заменяет
каждый пиксель **ссылкой на ближайший эталон** плюс небольшой поправкой
восстановления `(epsilon, delta_y, k_m)`.

Эталоны организованы **двухуровнево**:

```
Основной эталон 0 ─┬─ Под-эталон 0 (базовый)
                   ├─ Под-эталон 1
                   └─ Под-эталон 2

Основной эталон 1 ─┬─ Под-эталон 0 (базовый)
                   └─ Под-эталон 1
```

- **Уровень 1 (основные эталоны)** — крупные классы материалов. Сравнение идёт
  с базовым вектором (`[0]`) каждого основного эталона.
- **Уровень 2 (под-эталоны)** — мелкие вариации внутри выбранного класса.

## Поток данных при сжатии

Оркестрируется функцией `run_compress()` ([src/app.cpp](../src/app.cpp)):

```
Входной .gsd (int16, BIP)
    │
    ▼
read_hdr_file()  ─ метаданные из .hdr (samples, lines, bands, byte_order, …)
load_hsi_data()  ─ пиксели в матрицу [lines·samples][bands]
    │
    ▼
compression()        ([src/compression.cpp](../src/compression.cpp))
  для каждого пикселя:
    select_pixel()   ─ если >50% полос отрицательны → заменить нулевым вектором
    check_pixel()    ─ двухуровневый матчинг (см. ниже)
    add_standart_data() ─ дописать результат пикселя в image[] (массив с удвоением ёмкости)
    │
    ▼
[--write 1]  save_standarts_gsd()        ─ словарь эталонов → standarts.gsd
             save_compressed_image_gsd() ─ карта пикселей    → image.gsd
    │
    ▼
[--rgb 1]    save_rgb_image()  ─ превью оригинала → preview.png
             decompress()      ─ восстановление из памяти
             save_rgb_image()  ─ превью восстановленного → restored_preview.png
```

`main()` — тонкая точка входа: разбирает `argv` в структуру `cli_options` и
вызывает `run_compress()` / `run_decompress()`. Вся логика режимов — в
[src/app.cpp](../src/app.cpp).

### Отбраковка артефактов: `select_pixel`

Перед сжатием каждый пиксель проверяется: если больше половины спектральных
полос отрицательны (типичный артефакт датчика), пиксель **заменяется нулевым
вектором**. Пиксель не пропускается — он всё равно даёт запись в `image[]`, но
не порождает паразитных эталонов из шума.

## Двухуровневый матчинг: `check_pixel`

Реализован в [src/check_pixel.cpp](../src/check_pixel.cpp). Для каждого пикселя:

**0. Адаптивные пороги.** Вычисляется норма пикселя
`norm = pixel_norm(pixel, bands, method)` и пороги масштабируются под неё:

```
main_threshold       = main_error_pct       / 100 · norm
additional_threshold = additional_error_pct / 100 · norm
```

`pixel_norm` = второй момент `M²(y)` для НП/МП и дисперсия `D(y)` для ОП/АП —
та же величина, что стоит в знаменателе нормированной ошибки метода
(см. [MATH.md](MATH.md#адаптивные-пороги)).

**1. Уровень 1.** Среди базовых векторов `hsi_standarts[i][0]` всех основных
эталонов ищется ближайший (`find_nearest` — минимум `epsilon`).
- Если эталонов ещё нет (`best_i == -1`) или `best_r1.epsilon > main_threshold`
  → пиксель «не распознан» как класс → `create_main_standart()`:
  создаётся новый **основной** эталон.
- Иначе переходим к уровню 2.

**2. Уровень 2.** Внутри лучшего основного эталона `best_i` ищется ближайший
под-эталон среди всех `hsi_standarts[best_i][0..ref_counts-1]`.
- Если `best_r2.epsilon > additional_threshold` → `create_internal_standart()`:
  добавляется новый **под-эталон**.
- Иначе пиксель **назначается** найденному под-эталону: в результат пишутся
  `ref_index` (плоский индекс) и `match = {epsilon, delta_y, k_m}`.

### Плоская индексация под-эталонов

Под-эталоны адресуются сквозным **плоским индексом**:

```
ref_index = Σ ref_counts[0..i-1] + j
```

Функция `flat_index(cd, main_i, sub_j)` суммирует число под-эталонов всех
предыдущих основных эталонов и добавляет локальный `j`.

> **Важная деталь.** Когда `add_internal_standart` вставляет новый под-эталон в
> середину последовательности, все ранее записанные пиксели с
> `ref_index >= new_flat` получают `+1` к своему индексу — иначе ссылки «съехали
> бы». Это единственное место, где уже записанная карта пикселей правится задним
> числом.

## Смещённое размещение нового эталона

Когда пиксель `p` должен стать новым эталоном (на любом уровне), программа
сохраняет не сам пиксель, а его **смещённую копию**, отодвинутую от ближайших
существующих эталонов. Реализация — `make_shifted_etalon`
([src/etalon_shift.cpp](../src/etalon_shift.cpp)).

**Зачем.** Эталон, поставленный прямо в пиксель, своей зоной перекрывается с
зонами соседей — покрытие дублируется, а рядом остаются «дыры», требующие
лишних эталонов. Смещение расставляет эталоны ближе к идеальной решётке
покрытия: эталонов нужно меньше, сжатие выше. Вывод геометрии —
в [MATH.md](MATH.md#геометрия-покрытия-и-смещение-эталона).

**Как.** За один проход по кандидатам-соседям: для каждого соседа, чья ошибка
`epsilon` ниже целевой `threshold·sc`, текущий снимок пикселя отталкивается от
**метод-выровненного** соседа `êᵢ = k_m·eᵢ + Δy` вдоль направления в линейной
шкале `ρ = √(n·ε)` на величину `(target_ρ − ρ)`:

```
target_epsilon = threshold · sc
target_ρ       = √(bands · target_epsilon)
для каждого соседа i с 0 < εᵢ < target_epsilon:
    ρ     = √(bands · εᵢ)
    shift = target_ρ − ρ
    y ← y + (y − êᵢ)/ρ · shift          (репульсия от выровненного соседа)
```

- Аккумулятор сдвига хранится в `double` (полная точность между итерациями);
  для каждого сравнения берётся округлённый снимок int16.
- `sc` — коэффициент шага решётки (`settings.step_coef`, CLI `--sc`). При
  `sc = 1` все соседи нового эталона дальше радиуса зоны → вклад в сдвиг нулевой,
  механизм геометрически неактивен. Включается заданием `--sc > 1` (по теории
  до 2).
- На уровне 2 у смещения есть **якорь** (`anchor`): кандидат-сдвиг принимается,
  только если он остаётся в зоне основного эталона
  (`pixel_distance(anchor, y) ≤ main_threshold`). Так под-эталон не «убегает» из
  своего класса.
- Отключается полностью опцией `--shift 0` (`settings.shift_enabled`).

После создания смещённого эталона `set_match` записывает в `result->match`
фактическое соответствие `pixel_distance(e_new, p)` — чтобы декомпрессия
восстановила исходный пиксель из смещённой копии. При вырожденном результате
(NaN/inf) сохраняется тождественное соответствие `{0, 0, 1}`.

## Методы КЭКМ

Ядро матчинга — функция «расстояния» между спектрами. Реализованы четыре метода
с разной инвариантностью (выбор через `-m`):

| Метод | Флаг | Инвариантность | epsilon | Восстановление |
|-------|------|----------------|---------|----------------|
| **НП** | `-m 1` | — | `M²(e) + M²(y) − 2·M(ey)` | `ŷ = ref` |
| **ОП** | `-m 2` | ортогональные преобразования | `D(e) + D(y) − 2·cov(e,y)` | `ŷ = ref + Δy` |
| **МП** | `-m 3` | масштабирование | `M²(y) − M²(ey)/M²(e)` | `ŷ = k_m·ref` |
| **АП** | `-m 4` | аффинные преобразования | `D(y) − cov²(e,y)/D(e)` | `ŷ = k_m·ref + Δy` |

Каждый метод возвращает `kekm_result {epsilon, delta_y, k_m}`. Полный вывод
формул и смысл коэффициентов — в [MATH.md](MATH.md#методы-кэкм-вывод-формул).

## Форматы выходных файлов

### `standarts.gsd` — словарь эталонов

```
[int32]  total_sub_standards         — общее число под-эталонов
[int32]  bands                       — число спектральных полос
[int16 × total × bands]              — векторы эталонов, BIP, плоский порядок
```

Плоский порядок совпадает с `ref_index`: сначала все под-эталоны основного
эталона 0, затем основного эталона 1 и т.д.

### `image.gsd` — карта сжатых пикселей

```
[int32]  samples
[int32]  lines
на каждый пиксель (row-major):
  [int32]  ref_index    — плоский индекс эталона
  [double] epsilon      — оценка близости
  [double] delta_y      — смещение (ОП, АП)
  [double] k_m          — масштаб (МП, АП)
```

`bands` в `image.gsd` не дублируется — берётся из `standarts.gsd`. Размеры
изображения (`samples`/`lines`) самодостаточны, поэтому декомпрессия из GSD не
требует исходного `.hdr` (он нужен лишь для `byte_order`).

## Декомпрессия

Реализована в [src/decompress.cpp](../src/decompress.cpp) двумя путями.

### 1. Из памяти — `decompress()`

Используется сразу после сжатия в `run_compress()` для рендера
`restored_preview.png` — без файлового I/O. Принимает `compressed_image` в
памяти, строит плоский массив указателей на под-эталоны из 3D-структуры и
восстанавливает каждый пиксель.

### 2. Из бинарных GSD-файлов — `decompress_from_gsd_files()`

Входная точка подкоманды `decompress`. Читает `standarts.gsd` + `image.gsd`,
восстанавливает матрицу пикселей и пишет её через `save_hsi_data()` в `.dat`
(`--dat-out`, по умолчанию `restored.dat`) в той же раскладке int16/BIP, что и
входной `.gsd`. `samples`/`lines`/`bands` берутся из GSD-файлов и перезаписывают
значения заголовка; `.hdr` нужен только для `byte_order`.

### Универсальная формула восстановления

Обе ветки применяют к каждой полосе одно и то же обратное преобразование
(`apply_inverse`) с клиппингом в диапазон int16 `[−32768, 32767]`:

```
ŷ[b] = round(k_m · ref[b] + delta_y)
```

| Метод | k_m | delta_y | Результат |
|-------|-----|---------|-----------|
| НП | 1 | 0 | точная копия эталона |
| ОП | 1 | ≠0 | эталон + смещение по среднему |
| МП | ≠1 | 0 | масштабированный эталон |
| АП | ≠1 | ≠0 | полное аффинное восстановление |

Если `ref_index` вне диапазона словаря, пиксель остаётся нулевым (`calloc`).
