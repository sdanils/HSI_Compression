# HSI Compression

Программа для сжатия гиперспектральных изображений (ГСИ) на основе метода
квантования спектральных эталонов (КЭКМ).

Алгоритм заменяет каждый пиксель ссылкой на ближайший «эталон» (характерный
спектральный вектор) и небольшой поправкой для восстановления — вместо хранения
сырых спектральных данных. Это существенно сокращает объём при контролируемой
потере качества.

Подробное описание алгоритма, методов КЭКМ, форматов файлов и архитектуры —
в [docs/PROGRAM_DESCRIPTION.md](docs/PROGRAM_DESCRIPTION.md).

## Требования

- CMake >= 3.10
- C++17-совместимый компилятор (GCC, Clang, MSVC)

Внешних зависимостей нет: PNG-запись выполняется встроенной библиотекой
`include/stb_image_write.h`.

## Сборка

```bash
mkdir build && cd build
cmake ..
cmake --build .
```

Исполняемый файл: `build/hsi_compressor` (на Windows с MSVC —
`build/Debug/hsi_compressor.exe` или `Release/`).

## Входные данные

| Файл | Описание |
|------|----------|
| `.hdr` | Текстовый заголовок ENVI с метаданными (samples, lines, bands, byte order, длины волн) |
| `.gsd` | Бинарные данные пикселей (int16, интерлив BIP) |

## Запуск

Интерфейс построен на **подкомандах**: первый аргумент выбирает режим —
`compress` (сжатие) или `decompress` (восстановление).

```bash
# Сжатие (пути к .hdr и .gsd обязательны)
./hsi_compressor compress path/to/hsi.hdr path/to/hsi.gsd

# Сжатие с выбором метода АП и порогов
./hsi_compressor compress hsi.hdr hsi.gsd -m 4 --me 3.0 --ae 1.0

# Декомпрессия: восстановить .dat из GSD-файлов
./hsi_compressor decompress hsi.hdr --std standarts.gsd --img image.gsd --dat-out out.dat
```

### Основные опции

| Опция | Описание | По умолчанию |
|-------|----------|--------------|
| `-m, --method <1-4>` | Метод КЭКМ: 1=НП, 2=ОП, 3=МП, 4=АП | `1` (НП) |
| `-e, --me <%>` | Порог 1-го уровня (% от нормы пикселя) | `5.0` |
| `-a, --ae <%>` | Порог 2-го уровня (% от нормы пикселя) | `2.0` |
| `-S, --shift <0\|1>` | Смещённое размещение нового эталона | `1` |
| `-c, --sc <x>` | Коэффициент шага решётки: d = sc·δ, 1 ≤ sc ≤ 2 | `1.0` |
| `-r, --rgb <0\|1>` | Генерировать RGB-превью | `1` |
| `-w, --write <0\|1>` | Записывать выходные GSD-файлы | `1` |
| `-d, --dat-out <file>` | Выходной `.dat` (только `decompress`) | `restored.dat` |

Полный список опций — `./hsi_compressor compress` без аргументов или
в [docs/PROGRAM_DESCRIPTION.md](docs/PROGRAM_DESCRIPTION.md).

## Выходные данные

| Файл | Описание |
|------|----------|
| `preview.png` | RGB-превью оригинала (гауссов blend 660/550/470 нм + гистограммное выравнивание); отключается `--rgb 0` |
| `standarts.gsd` | Бинарный файл эталонов (`--std`) |
| `image.gsd` | Карта сжатых пикселей: ref_index + параметры восстановления (`--img`) |
| `restored_preview.png` | RGB-превью восстановленного изображения (тот же LUT, для визуального сравнения) |
| `restored.dat` | Восстановленная матрица пикселей — только в режиме `decompress` (`--dat-out`) |

## Алгоритм сжатия

Двухуровневое сопоставление в [src/check_pixel.cpp](src/check_pixel.cpp) с
**адаптивными порогами** (процент от нормы самого пикселя):

1. **Уровень 1 (основные эталоны):** пиксель сравнивается с базовым вектором
   каждого основного эталона. Если близость хуже порога `--me` — создаётся
   новый основной эталон.
2. **Уровень 2 (под-эталоны):** внутри подходящего основного эталона пиксель
   сравнивается со всеми под-эталонами. Если близость хуже порога `--ae` —
   добавляется новый под-эталон.

Близость измеряется одним из четырёх методов КЭКМ (НП/ОП/МП/АП) с разными
свойствами инвариантности — см. [docs/PROGRAM_DESCRIPTION.md](docs/PROGRAM_DESCRIPTION.md).

Новый эталон по умолчанию сохраняется не как сам пиксель, а как его **смещённая
копия**, отодвинутая от соседних эталонов ближе к идеальной решётке покрытия —
это уменьшает число эталонов (управляется `--shift` и `--sc`).

Более высокие пороги → меньше эталонов → сильнее сжатие, ниже качество.

## Структура проекта

```
HSI_Compression/
├── src/
│   ├── main.cpp          # точка входа: разбор argv и выбор режима
│   ├── app.cpp           # режимы run_compress / run_decompress
│   ├── compression.cpp   # основной цикл сжатия
│   ├── check_pixel.cpp   # двухуровневое сопоставление с эталонами
│   ├── etalon_shift.cpp  # смещённое размещение нового эталона
│   ├── kekm.cpp          # четыре метода КЭКМ
│   ├── decompress.cpp    # восстановление (из памяти и из GSD-файлов)
│   ├── load_hsi_data.cpp # загрузка бинарных .gsd данных
│   ├── read_hdr_file.cpp # разбор .hdr заголовка
│   ├── save.cpp          # запись GSD-файлов
│   ├── save_rgb.cpp      # RGB-превью (PNG)
│   └── free_memory.cpp   # освобождение памяти
├── include/              # заголовки модулей (functions.h — зонтичный)
├── docs/                 # подробная документация
└── CMakeLists.txt
```
