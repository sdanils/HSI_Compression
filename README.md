# HSI Compression

Программа для сжатия гиперспектральных изображений (ГСИ) с помощью двухкритериального алгоритма распознавания с самообучением.

Алгоритм заменяет каждый пиксель ссылкой на ближайший «эталон» вместо хранения сырых данных, что позволяет существенно сократить объём хранимой информации при контролируемой потере качества.

## Требования

- CMake >= 3.10
- C++17-совместимый компилятор (MSVC, GCC, Clang)

## Сборка

```bash
mkdir build && cd build
cmake ..
cmake --build .
```

На Windows с MSVC исполняемый файл появится в `build/Debug/hsi_compressor.exe` (или `Release/`), на Linux/macOS — в `build/hsi_compressor`.

## Входные данные

Программа ожидает два файла:

| Файл | Описание |
|------|----------|
| `hsi.hdr` | Текстовый заголовок с метаданными изображения (формат ENVI HDR) |
| `hsi.gsd` | Бинарные данные пикселей (int16, формат BIP) |

По умолчанию пути заданы как `/.data/hsi.hdr` и `/.data/hsi.gsd`. Передайте свои пути через аргументы командной строки (см. ниже).

## Запуск

```bash
# Использование путей по умолчанию
./hsi_compressor

# Указать пути к файлам явно
./hsi_compressor path/to/hsi.hdr path/to/hsi.gsd

# Указать пути и переопределить параметры из заголовка
./hsi_compressor path/to/hsi.hdr path/to/hsi.gsd <samples> <lines> <byte_order>
```

| Аргумент | Описание |
|----------|----------|
| `hdr_path` | Путь к файлу заголовка `.hdr` |
| `dat_path` | Путь к бинарному файлу данных `.gsd` |
| `samples` | Ширина изображения (количество столбцов) |
| `lines` | Высота изображения (количество строк) |
| `byte_order` | Порядок байтов: `0` = little-endian, `1` = big-endian |

## Выходные данные

| Файл | Описание |
|------|----------|
| `rgb_preview.png` | RGB-превью изображения (каналы R/G/B выбраны по длинам волн 660/550/470 нм) |
| `standarts.txt` | Список всех эталонов (при включённом сжатии) |
| `image.txt` | Таблица сопоставления пикселей с эталонами: `main_ref`, `internal_ref`, `MSE` |

## Алгоритм сжатия

Двухуровневое сопоставление в [check_pixel.cpp](src/check_pixel.cpp):

1. **Уровень 1 (основные эталоны):** Вычисляется MSE между пикселем и каждым основным эталоном. Если `MSE > main_error` — создаётся новый основной эталон.
2. **Уровень 2 (внутренние эталоны):** Внутри наилучшего основного эталона проверяются внутренние под-эталоны. Если `MSE > additional_error` — создаётся новый внутренний эталон.

### Настройка порогов ошибки

Пороги задаются в [src/main.cpp](src/main.cpp) через структуру `compression_settings`:

```cpp
compression_settings settings = {26000, 21000};
//                                ^      ^
//                          main_error  additional_error
```

| Пороги | Результат |
|--------|-----------|
| Высокие | Меньше эталонов → меньше файл, ниже качество |
| Низкие  | Больше эталонов → больше файл, выше качество |

## Структура проекта

```
HSI_Compression/
├── src/
│   ├── main.cpp               # Точка входа
│   ├── check_pixel.cpp        # Двухкритериальное сопоставление
│   ├── calculate_mse.cpp      # Вычисление MSE
│   ├── load_hsi_data.cpp      # Загрузка бинарных данных
│   ├── read_hrd_file.cpp      # Разбор .hdr заголовка
│   ├── save.cpp               # Сохранение эталонов и маппинга
│   └── save_rgb.cpp           # Сохранение RGB-превью
├── include/
│   ├── functions.h            # Документация всех функций
│   ├── hsi_header.h           # Структура заголовка HSI
│   ├── compressed_image.h     # Структура сжатого изображения
│   ├── compression_settings.h # Настройки порогов ошибки
│   └── standart_data.h        # Результат сопоставления пикселя
└── CMakeLists.txt
```

Подробная документация по каждой функции — в [include/functions.h](include/functions.h).
